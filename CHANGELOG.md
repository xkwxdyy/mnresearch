## v0.1.18 (2026-03-14 17:10:00)

### ✨ 策略编辑升级

#### 策略终于能同时带多个推进方式标签
- 策略编辑弹窗里的单一“策略类型”升级成了可复用的多标签输入，支持回车/逗号收标签、历史召回和删除标签

- 策略列表、分支地图、Focus 候选等展示位都会显示多枚推进方式标签，不再把组合打法硬压成一个词

### 🧭 术语整理

#### 策略结构和结果闭环终于说人话了
- 策略字段文案改成 `推进方式（可多选）` 和 `策略角色`，把“用了什么招”和“在整体布局里扮演什么角色”拆开
- 围绕子问题闭环的文案统一收口到 `分支 / 回流`
- `关联` / `子问题` 用词重新统一，撤掉了这次改动里不顺耳的 `挂接` / `子题`

## v0.1.17 (2026-03-14 16:11:55)

### 🐛 Markdown 渲染补漏

#### 研究内容终于不再一半富文本一半生字符
- 补齐动作卡片、当前 Focus 绑定、策略拆题按钮、悬空子题按钮、文献目标提示等残留展示位的 Markdown 渲染，用户写进去的加粗、列表、公式不再随机失效
- 时间线元信息现在会对关联的问题、判断、策略、障碍、样例、认识、文献标题走富文本渲染，不再把相关对象标题硬退回转义文本
- 删除 / 编辑弹窗里的对象标题、挂接副标题、影响卡片值、文献目标摘要等也统一走 Markdown，避免同一段研究内容在主界面能渲染、进弹窗又失真
- 含问题标题的 HTML 下拉选项补上 `labelHtml`，自定义下拉里不再把研究问题标题降级成纯文本


## v0.1.16 (2026-03-14 16:00:27)

### ✨ 内容编排

#### 策略、判断、样例、障碍、认识终于能调顺序了
- 问题工作台里的策略、判断、样例、障碍、认识列表都新增了上移 / 下移操作，不再只能按追加顺序硬扛
- 这些内容现在都会稳定持久化 `order`，旧数据读入时也会自动补齐顺序，避免“看起来有顺序字段，实际没人管”的半成品状态
- 总览、分支地图和问题关系读取统一改成按内容顺序取值，当前策略/判断/认识不会再被数组残余顺序偷偷带偏

### 🐛 时间线修复

#### 子问题回流终于不再只写进抽屉
- 子问题保存“回流”时，会同时在子问题和母问题各写一条系统时间线事件，不再只改状态字段
- 母问题时间线里的这条事件会直接关联到对应子问题，点开就能看见回流状态变更和摘要
- 母问题“策略与子问题”分支地图现在会直接显示每个绑定子问题的回流摘要，主线不再断成黑箱

#### 时间线详情终于能点开了
- 修复时间线事件点击后被自己立刻清空弹窗状态的问题；现在点任意一条都会稳定打开详情弹窗，不再只闪一下装死

#### 原生状态层别再把时间线当空气
- 修复 native 状态归一化漏掉 `timelineEvents` / `actions` / `focusState` 的致命问题；旧状态加载和新状态保存现在会保留时间线、动作和 Focus 绑定，不会再在原生层被静默洗掉

#### 日期终于不再装死
- 时间线列表与详情里的时间戳现在统一显示完整日期和时间，不再只露出 `HH:mm` 让跨天记录看起来像串到一起
- 时间线排序改成按真实时间戳比较，不再依赖字符串字面排序，记录顺序会稳定绑定到 `createdAt` 对应的日期时间

#### 时间线正文开始认真渲染 Markdown
- 时间线事件现在支持把首行以下的正文作为详情区块渲染，列表和详情弹窗都会按 Markdown 显示
- 时间线详情弹窗里的“旧值 / 新值”改成 Markdown 渲染，问题表述、策略说明这类用户内容不再退回成转义后的纯文本

## v0.1.13 (2026-03-13 14:41:19)

### 🎉 重大新功能

#### Focus 视图与时间线系统
- **Focus 视图**：新增独立的研究聚焦视图，支持绑定问题、策略或动作作为当前聚焦对象
  - Focus 状态独立于问题树导航，保持上次绑定记忆
  - 支持快速记录进展、认识、卡点、下一步等时间线事件
  - 动作区默认收起，展开后提供"设为当前聚焦"、"改状态"、"查看详情"等操作
  - 支持"仅看当前聚焦对象"和"包含子问题"的过滤选项
- **时间线系统**：新增完整的研究时间线功能，自动记录研究过程中的所有关键事件
  - 系统自动记录：问题、判断、策略、障碍、认识的新增、编辑、状态变更
  - 手动快速记录：支持记录进展、认识、卡点、下一步等四类事件
  - 时间线详情弹窗：点击事件查看详情，支持编辑、删除、升格操作
  - 事件升格：支持将时间线事件升格为认识或障碍，并自动记录升格关系
  - 时间瀑布流设计：时间优先显示，第一行"时间+类型+摘要"，第二行"来源+关联对象"
- **工作区模式切换**：新增三种工作区模式
  - Dashboard：研究驾驶舱，全局概览
  - Workbench：问题工作台，深度工作
  - Focus：聚焦视图，专注当前任务

### ✨ 功能增强

#### 动作状态管理
- 动作状态采用五态模型：`queued`（排队）、`doing`（进行中）、`waiting`（等待）、`blocked`（受阻）、`done`（完成）
- 动作卡片支持快速改状态，直接走小菜单，不再多跳一步
- 支持将动作设为当前聚焦对象

#### 时间线过滤与筛选
- 新增双下拉过滤器：类型过滤（全部/进展/认识/卡点/下一步）+ 来源过滤（全部/手动输入/系统记录）
- 进展事件可单独筛出，方便对导师汇报
- 类型和来源分开，不混成一个大筛选器

#### 默认视图调整
- 选中问题后默认进入时间线视图（timeline），不再是总览（overview）
- 新建问题后默认进入问题表述页（formulation）
- 研究驾驶舱默认显示时间线视图

### 🔧 技术改进

#### 系统事件记录机制
- 实现 `recordSystemEvent` 函数，统一处理系统事件记录
- 实现 `recordManualFocusEvent` 函数，处理手动快速记录
- 实现 `buildDiffSummary` 函数，自动生成变更摘要（"标签：旧值 -> 新值"）
- 所有关键操作（新增、编辑、状态变更）自动触发时间线记录

#### Focus 状态管理
- 实现 `getFocusState`、`buildFocusState`、`setFocusState` 函数
- Focus 状态包含：questionId、entityType、entityId
- 支持策略和动作作为聚焦对象，自动验证对象存在性和归属关系

#### 数据模型扩展
- 时间线事件支持 `promotedEntityType` 和 `promotedEntityId` 字段，记录升格关系
- 新增 `UPDATE_FOCUS_STATE`、`ADD_TIMELINE_EVENT`、`UPDATE_TIMELINE_EVENT`、`DELETE_TIMELINE_EVENT` 等 action
- 新增 `timelineEventModalId`、`focusActionsExpanded`、`focusOnlyCurrentEntity`、`focusIncludeChildQuestions` 等 UI 状态

### 📚 文档更新

#### Focus 与时间线重设计文档
- 新增 8 个设计决策章节（1.11-1.18）
- 明确时间线详情用按需弹窗，不做常驻详情栏
- 明确时间线列表要有"时间瀑布"感
- 明确当前聚焦只允许策略和动作
- 明确动作区保留但默认收起
- 明确动作状态采用五态
- 明确进展不升格但必须可单独筛出
- 明确过滤器采用两个下拉
- 明确 Focus 与问题树是独立状态
